# 予測信頼度（Prediction Reliability）の仕組み

## 1. 前提: MLモデルの学習方法（5-Fold CV）

モデルは **5-Fold Cross Validation** で学習している。データを5分割し、4つで学習→1つで検証を5回繰り返す。

```
Fold 1: [学習][学習][学習][学習][検証]  → モデル1
Fold 2: [学習][学習][学習][検証][学習]  → モデル2
Fold 3: [学習][学習][検証][学習][学習]  → モデル3
Fold 4: [学習][検証][学習][学習][学習]  → モデル4
Fold 5: [検証][学習][学習][学習][学習]  → モデル5
```

つまり **5つの異なるモデル** が生成される。

## 2. fold_predictions: 各モデルの全データ予測

各foldのモデルで **全データ** を予測する。

`src/analyze/ml_model.py` の `train_cv` 関数内:

```python
full_pred = model.predict(features)  # fold毎のモデルで全行を予測
shuffled_fold_predictions[:, fold_idx - 1] = full_pred
```

結果は `(9665行 × 5fold)` の行列になる。各行（＝各メッシュ×ジャンル）に対して5つの予測値が得られる。

**注意**: fold_predictionsは信頼区間の計算にのみ使用される。スコアリングに使う `oof_predictions` は各foldのバリデーションセットのみの予測であり、データリークはない。

## 3. gap_std: fold間のばらつき

`compute_market_gap` 関数内:

```python
gap_std = np.std(fold_predictions, axis=1)
```

5つのモデルの予測値の **標準偏差** を計算する。

- **gap_stdが小さい** → どのfoldで学習しても似た予測 → モデルが自信を持っている
- **gap_stdが大きい** → foldによって予測がバラバラ → 学習データへの依存が高い

## 4. 95%信頼区間（CI）

```python
gap_ci_lower = market_gap - 1.96 * gap_std
gap_ci_upper = market_gap + 1.96 * gap_std
```

`market_gap`（予測log店舗数 - 実際log店舗数）に対する95%信頼区間を算出する。この区間が **ゼロを跨ぐかどうか** が信頼度判定の基準になる。

## 5. gap_reliability: 信頼度ラベル（CI跨ぎベース）

95%信頼区間がゼロを跨ぐかどうかで3段階に分類する。これは統計的仮説検定の考え方に基づく手法であり、データに依存した閾値チューニングが不要という利点がある。

```python
reliability = "low"   # デフォルト

# high: CI下限 > 0 → 95%信頼で「予測 > 実際」と言える
if ci_lower > 0:
    → "high"

# medium: gap > 0 だがCIが0を跨ぐ → 正のギャップだが統計的に不確実
elif market_gap > 0:
    → "medium"

# low: gap <= 0 → モデル上は出店余地なし
else:
    → "low"
```

| 信頼度 | 条件 | 意味 |
|--------|------|------|
| high | CI下限 > 0 | 統計的に有意な出店余地。5モデルとも「予測 > 実際」と一致 |
| medium | gap > 0 かつ CI下限 ≤ 0 | 出店余地の可能性はあるが、モデル間でばらつきがあり不確実 |
| low | gap ≤ 0 | モデル上は出店余地なし。予測店舗数 ≤ 実際の店舗数 |

### 設計の経緯

当初は `gap_std` の絶対値に閾値を設ける方式（high: ≤0.1, medium: ≤0.2）を採用していたが、以下の問題があった:

- データの gap_std 中央値=0.036, 75%ile=0.063 に対して閾値が緩すぎ、high=71.3%, low=1.0% と極端に偏った
- 閾値がデータ固有で、再学習のたびに妥当性を検証する必要があった

CI跨ぎベース（方法2）に変更することで:
- 閾値のチューニングが不要（統計的な意味で自動的に決まる）
- データが変わっても判定基準の意味が一貫する
- 「出店余地あり / 不確実 / なし」の3分類が直感的

## 7. 推薦フィルタでの活用

### CLIスクリプト（scripts/train_model.py）

`filter_recommendations` 関数で信頼度の低い推薦を除外する:

```python
filter_recommendations(df, min_rc=2, min_gap_count=1.0, min_reliability="medium")
```

- `min_rc=2`: 現在2店舗以上あるエリアのみ（データが少なすぎる場所を除外）
- `min_gap_count=1.0`: 予測と実際の差が1店舗以上
- `min_reliability="medium"`: high + medium のみ（lowを除外）

### ダッシュボード（src/visualize/dashboard.py）

「予測信頼度」ドロップダウンで3段階のフィルタを提供:

| 選択肢 | フィルタ内容 |
|--------|-------------|
| すべて | フィルタなし（全データ表示） |
| 高+中（出店余地あり） | high + medium のみ表示（low を除外） |
| 高のみ（統計的に有意） | high のみ表示 |

地図表示・ランキングテーブルの両方に連動する。

## 8. ダッシュボードでのポップアップ表示

メッシュ 5339465211 × イタリアン:

  market_gap=+0.15  gap_std=0.10

  market_gap=-0.30
