# 市場空白地帯発見システム (Market Gap Finder) システム概要メモ

## 1. システムの目的とコンセプト
このシステムの最大の目的は、**「需要が高いにもかかわらず競合が少ないエリア（空白地帯）」を特定すること**です。
「どのエリア（例: 緯度経度のメッシュ）」に「どのジャンル（例: 居酒屋、カフェ）」を出店するのが最も勝率が高いかを、**機会スコア（Opportunity Score）** として数値化し、ランキング形式でレコメンドします。

## 2. コアとなる分析ロジック（スコアリング）
システムの根幹は、需要と供給のバランスを評価する以下の計算式です。

**`機会スコア = (需要の重み × 需要指標) − (供給の重み × 供給指標)`**

* **需要指標 (Demand Index):** そこに飲食店を求める人がどれくらいいるか。
* **供給指標 (Supply Index):** 既に同ジャンルの競合店舗がどれくらい存在するか。

### スコアリングの進化 (v1 → v2)
現在、システムはより高度なフェーズ（v2）へ移行中です。
* **v1コア:** 主に口コミ数や評価（Google Places）などを需要とし、同ジャンル店舗の密度（HotPepper / Google Places）を供給としていました。
* **v2コア (最新):** 国勢調査のオープンデータである **e-Stat（政府統計ポータル）** と連携し、**「JIS第3次メッシュ単位（約1km四方）の詳細な人口データ」を需要のベース** として組み込むパイプラインが構築されました。これにより、より実態に即した精緻な需要予測が可能になっています。

## 3. データパイプラインと処理フロー
直近実装されたパイプライン（`scripts/integrate_estat.py`）を中心に、データはどう流れ、処理されるのかを説明します。

1. **データ収集 (Collect):** HotPepper API 等から特定エリア（例えば東京）の店舗データを取得。
2. **前処理 (Preprocess):** 飲食店のジャンルを正規化し、店舗の緯度経度から「JIS第3次メッシュコード」を算出して割り当てます。
3. **外部連携・統合:** e-Stat API から取得した「国勢調査メッシュ人口データ」と、HotPepper側の既存店舗データ（供給データ）を「3次メッシュコード」をキーにして結合します。
4. **スコアリング (Analyze):** 上記ステップで作られた「人口データ（需要）× 競合データ（供給）」の統合データに対して、v2の機会スコアを計算します。
5. **出力:** スコアが高い上位候補をリストアップし、加工済みデータとして保存（`data/processed/tokyo_integrated.csv`）します。

## 4. システムの構造（アーキテクチャ）
役割ごとに明確にコンポーネントが分割されています。

* **`src/collect/` (収集層):** 
  * `hotpepper.py` / `google_places.py`: 飲食店の競合情報などを取得。
  * `estat.py`: e-Stat APIと通信し、統計表やメッシュ人口を取得。（新規追加）
* **`src/preprocess/` (前処理層):** 
  * `cleaner.py`: データクレンジングや名寄せ処理。
  * `mesh_converter.py`: 緯度経度とメッシュコードの相互変換。（新規追加）
* **`src/analyze/` (分析層):**
  * `scoring.py`: メインのスコア算出ロジック。
  * `backtest.py`: 過去データを用いた予測テスト機能。
* **`src/visualize/` (可視化層):**
  * `dashboard.py`: Foliumを用いた地図表示やPlotly Dashによるダッシュボード構築。
* **その他:**
  * `scripts/`: データパイプラインを回すためのCLIスクリプト群。
  * `notebooks/`: データ探索・基礎分析（EDA）を行うためのJupyter Notebook群。

## 5. 今後の拡張ロードマップ（Next Steps）
現在、システムの基盤とe-Statデータの結合までが完了しており、今後は以下のステップ（P1〜P4）でシステムを高度化していく計画になっています。

* **検証と可視化 (P1/P2):** 
  v2スコア（人口データ反映版）の妥当性を深堀りし、地図上で人口ヒートマップと一緒に出店候補地を見比べられるように、ダッシュボード機能をアップデートします。
* **モデルの高度化 (P3):** 
  単に「人口」と「店舗数」だけでなく、「最寄り駅からの距離」「他商業施設の密度」「人口増減のトレンド（時系列）」といった空間的・時間的特徴量を追加し、将来的には LightGBM 等の機械学習モデルを導入して予測精度を上げる予定です。
* **バックテスト検証 (P4):** 
  実際の「成功した店舗・撤退した店舗」の過去データをシステムに読み込ませ、このシステムが出す機会スコアと実際の勝率に相関があるか（本当に使えるシステムか）を評価します。
